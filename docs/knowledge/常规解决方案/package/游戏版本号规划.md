## 游戏版本号

定义：用来控制游戏内容的热更、强更

<DocImage src='version/v1.png'></DocImage>

X：大版本迭代，例如：新增2、3个新系统上线版本。

Y：版本迭代，例如：1个月常规强更新版本，优化系统，小功能迭代等。

Z：预留容错版本号。例如：版本上线后需要有换包修复的问题需要强制换包预留版本号。

W：热更版本号。控制客户端patch包版本号。

版本号格式 X.Y.Z.W

X.Y.Z用于强更，W用于热更。



## 提审版本号

定义：用来进行IOS和Google的提审的

<DocImage src='version/v2.png'></DocImage>

X.Y.C （X.Y表示策划的大版本规划，C用来进行提审的版本号，每提审一次，自动+1，和游戏版本号的Z意义不一样）

D （对应 Android 中的 VersionCode，每次提审+1，更新版本不重置；对应 iOS 中的 CFBundleVersion，每次上传 TF +1，更新版本不重置）
